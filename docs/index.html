FOCUS: Action-Conditioned Partial Rollout for World Action Models

TL;DR

FOCUS updates only action-relevant local latent tokens during World Action Model rollout, preserving the benefit of future-latent inference while avoiding dense full-scene transition.

Abstract

World Action Models (WAMs) promise to improve robot control by grounding actions in predicted future scene states. However, dense future synthesis updates the full latent scene, while many manipulation decisions depend on only a small set of local variables. FOCUS addresses this mismatch with an action-conditioned partial rollout framework. It predicts the local support relevant to the current action, applies learned transition only on this support, and carries the remaining tokens through an identity path before action generation.

The selector is distilled from dense rollout teachers using pseudo-labels that require both teacher latent change and forward or inverse action relevance. We instantiate FOCUS in the Joint and IDM rollout paths of Fast-WAM. On LIBERO, FOCUS improves Fast-WAM-Joint from 93.0% to 96.4% success and Fast-WAM-IDM from 92.3% to 96.8%, while reducing mean inference latency from 0.66s to 0.45s and from 0.95s to 0.68s, respectively.

Core Question

Q. If a WAM performs future-latent rollout for control, does it need to update the entire scene?

A. No. Updating action-relevant local latent state is sufficient to recover the useful control signal in rollout-based Fast-WAM variants.

Method

FOCUS uses the support rule D ∩ (F ∪ I). The change gate D selects tokens changed by the dense teacher, while F and I identify forward and inverse action relevance. A learned selector predicts this support at inference time, and a transition head predicts residual updates only for selected tokens.

Results

We evaluate on all four LIBERO suites. Each suite contains 10 tasks, and each task is evaluated with 100 closed-loop trials. FOCUS improves both rollout-based Fast-WAM variants while reducing inference latency.
